# Stellar Trusted Address Registry

Реестр доверенных Stellar-адресов участников MTL Programmers Guild.

> **Важно:** on-chain поля (Name, TelegramUserID) — самодекларация. Надёжная верификация = перекрёстные факты (signer-позиция, взаимные ссылки, история транзакций, подтверждение через независимый канал).

## Регламент верификации

### Уровни доверия
- **trusted** — несколько независимых факторов подтверждения
- **known** — один надёжный фактор (например, signer в мультисиге)
- **unverified** — самодекларация только

### Что считается надёжным доказательством
1. Адрес стоит в signers мультиподписного аккаунта (добавлялся по договорённости)
2. Взаимные ссылки: A → B и B → A на блокчейне (сложнее подделать одновременно)
3. История транзакций совпадает с ожидаемой
4. Подтверждение через независимый канал (Grist mailbox, Telegram от известного ID)

### Пороги действий
- **Known signer + небольшая сумма** → принять автоматически
- **Unknown address** → уточнить через mailbox перед ответным действием
- **Unknown + крупная сумма** → отклонить до верификации

### Процедура проверки нового адреса
1. Проверить в LORE: https://lore.mtlprog.xyz/api/v1/accounts/АДРЕС
2. Проверить home_domain через Stellar Expert
3. Если MTL/Montelibero — проверить через mtla.me или у Стаса
4. Подтверждение через второй канал (Telegram, Grist mailbox)
